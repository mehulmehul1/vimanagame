## Key Principles

1. **Visual First** - Everything that can be visualized should be (states, timelines, triggers)
2. **Iterative** - Play-in-editor allows rapid testing without full builds
3. **Non-Destructive** - Changes in editor don't break the game
4. **Criteria System** - Your existing MongoDB-style queries become visual blocks
5. **Timeline-Based** - All choreography uses a unified timeline concept
6. **Branching** - Dialog and state transitions naturally support branches
